# TikTok Studio Analytics Exporter — Chrome Extension Plan

## Purpose

A Chrome extension that lets Filipino micro-creator participants in the thesis study export their TikTok Studio analytics to a CSV file with one click. Eliminates manual transcription of per-video metrics needed for the dataset.

## Confirmed API Endpoints (from DevTools inspection)

### Video List API
```
GET https://www.tiktok.com/tiktok/creator/manage/item_list/v1/
  ?locale=en&aid=1988&priority_region=PH&...&cursor=0
```
- Returns `item_list[]` with `aweme_id`, `create_time` (Unix timestamp), `comment_count`, `cover_url` per video
- Paginated: 50 videos per page, cursor-based (`has_more: true/false`)
- **Caveat**: URL includes `X-Bogus` and `X-Gnarly` anti-bot signatures generated by TikTok's own JS. The extension cannot forge these — use passive interception only (capture responses TikTok Studio fires naturally).

### Insight API
```
GET https://www.tiktok.com/aweme/v2/data/insight/
  ?locale=en&aid=1988&...
  &type_requests=[{"insigh_type":"video_retention_rate_realtime","aweme_id":"..."},...]
```
- Note: TikTok typo — parameter is `insigh_type` (missing 't'), not `insight_type`
- No `X-Bogus`/`X-Gnarly` in URL — direct fetch calls from content script should work
- Returns all per-video metrics in a single response per video

### Fields Extractable from Insight API (per video)

| CSV Column | JSON Path | Notes |
|---|---|---|
| `video_id` | `video_info.aweme_id` | |
| `post_date` | `video_info.create_time` | Unix timestamp → ISO date |
| `caption` | `video_info.desc` | |
| `duration_ms` | `video_info.video.duration` | milliseconds |
| `views` | `realtime_total_video_views.value.value` | |
| `likes` | `video_info.statistics.digg_count` | |
| `comments` | `video_info.statistics.comment_count` | |
| `shares` | `video_info.statistics.share_count` | |
| `ECR` | `video_retention_rate_realtime.value.list` where `timestamp=="5000"` → `.value` | e.g. 0.46 |
| `avg_watch_time_s` | `video_per_duration_realtime.value.value` | seconds |
| `NAWP` | computed: `avg_watch_time_s / (duration_ms / 1000)` | |
| `watched_full_pct` | `video_finish_rate_realtime.value.value` | e.g. 0.1273 |
| `traffic_foryou_pct` | `video_traffic_source_percent_realtime.value.value[key=="For You"].value` | |
| `traffic_follow_pct` | same, `key=="Follow"` | |
| `traffic_profile_pct` | same, `key=="Personal Profile"` | |
| `traffic_search_pct` | same, `key=="Search"` | |
| `new_followers` | `video_new_followers.value` | |
| `creator_uid` | `video_info.author.uid` | same across all videos |
| `creator_handle` | `video_info.author.unique_id` | same across all videos |
| `follower_count` | from user API call (see below) | current count at export time |
| `account_created_date` | from user API call (see below) | |

### User/Creator API (one call per export session)
```
GET https://www.tiktok.com/aweme/v1/user/profile/self/
  ?locale=en&aid=1988&...
```
- Seen in network tab as `user?aid=1988`
- Returns `follower_count` and account `create_time` (account age)
- Called once at start of export, values appended to every row

## Extension Architecture

### Files
```
tiktok-analytics-exporter/
├── manifest.json          # Manifest V3
├── background.js          # Service worker: network interception + insight API calls
├── content.js             # Page detection, floating UI injection
├── popup.html             # Extension popup: date picker + controls
├── popup.js               # Popup logic
└── icons/
    ├── icon16.png
    ├── icon48.png
    └── icon128.png
```

### manifest.json permissions
```json
{
  "manifest_version": 3,
  "name": "TikTok Studio Analytics Exporter",
  "version": "1.0",
  "permissions": ["webRequest", "storage", "tabs", "scripting"],
  "host_permissions": ["*://www.tiktok.com/*"],
  "background": { "service_worker": "background.js" },
  "content_scripts": [{
    "matches": ["*://www.tiktok.com/creator-center/content*"],
    "js": ["content.js"]
  }],
  "action": { "default_popup": "popup.html" }
}
```

## User Flow

```
1. Creator opens TikTok Studio video list
   → URL: tiktok.com/creator-center/content/

2. Creator clicks extension icon
   → Popup shows date range picker (Start date / End date)
   → Quick presets: "Last 30 days", "Last 3 months"

3. Creator clicks "Start Export"
   → Popup shows progress bar: "Fetching video list..."

4. Extension Phase A — Video List Collection (passive interception)
   → background.js intercepts item_list/v1 responses as TikTok Studio loads them
   → content.js injects a scroll-to-bottom script to trigger TikTok's natural pagination
   → For each intercepted response: extract aweme_id + create_time, filter by date range
   → Repeat until has_more == false
   → Progress: "Found N videos in date range"

5. Extension Phase B — Insight API calls (direct fetch)
   → For each aweme_id in the filtered list:
       - content.js calls fetch("https://www.tiktok.com/aweme/v2/data/insight/?...&type_requests=[...]")
       - type_requests includes all needed insigh_types (see list above)
       - Parse response, extract all fields, compute NAWP
       - Delay 2000ms between calls to avoid rate limiting
   → Progress bar updates: "Processing video X of N..."

6. Extension Phase C — Creator profile (direct fetch, once)
   → Fetch user profile API to get follower_count and account_create_time
   → Append to every row

7. Export
   → "Download CSV" button appears
   → All rows written as CSV blob, downloaded to creator's machine as
     tiktok_analytics_{handle}_{date}.csv
   → Creator sends this file to researcher
```

## Data Flow Diagram

```
TikTok Studio page
    │
    ├─ item_list/v1 responses (TikTok fires naturally on scroll)
    │       └─► background.js intercepts → filters by date → queue of aweme_ids
    │
    └─ content.js triggers scroll to bottom (forces pagination)

For each aweme_id in queue:
    content.js → fetch(insight API) → parse → accumulate rows

Once done:
    content.js → fetch(user profile API) → add creator fields to all rows

Export:
    popup.js → triggers CSV download from accumulated rows
```

## Error Handling

| Scenario | Behaviour |
|---|---|
| Video has insufficient data (`status: 2`) | Row included with empty fields for that metric; flagged in `data_quality` column |
| Insight API call fails (network error) | Retry once after 3s; if still fails, skip video and log video_id to console |
| `video_retention_rate_realtime` has no entry for timestamp 5000 | ECR left empty (video shorter than 5s, or data unavailable) |
| `has_more` never becomes false | Cap at 2000 videos; show warning |
| X-Bogus validation fails on insight API | Fall back: open each video analytics page in a hidden tab, intercept the insight response TikTok fires naturally |

## Known Risks and Mitigations

| Risk | Mitigation |
|---|---|
| TikTok changes item_list/v1 or insight API URLs | URL matching uses wildcard patterns; update in manifest host_permissions |
| X-Bogus required on insight API too | Fall back to page-navigation + passive interception (slower but always works) |
| Rate limiting on insight API | 2s delay between calls; exponential backoff on 429 responses |
| Retention data unavailable for very new videos (< 24h old) | `is_data_delay: true` flag in response; skip or mark as pending |
| TikTok detects scroll injection as bot | Use `window.scrollTo` with smooth behavior; randomise delay ±500ms |
| Creator has > 2000 videos | Warn user; date range filter limits the queue to a manageable subset |

## Distribution (Manual Developer Mode)

Creator install steps (include in participant kit):
1. Download and unzip `tiktok-analytics-exporter.zip` (sent by researcher)
2. Open Chrome → go to `chrome://extensions`
3. Enable "Developer mode" (toggle, top right)
4. Click "Load unpacked" → select the unzipped folder
5. Pin the extension to the toolbar (puzzle icon → pin)
6. Navigate to TikTok Studio → Content tab → click the extension icon

Note: Chrome shows a "You have developer extensions enabled" banner. Tell participants this is expected and safe.

## Out of Scope

- Submitting data directly to a server (privacy — all data stays local until creator sends the CSV)
- Firefox/Edge support (Manifest V3 differences not worth the effort for a research tool)
- Video thumbnail or MP4 download (separate yt-dlp workflow)
- Analytics for photo/carousel posts (only video posts are in scope for thesis)
